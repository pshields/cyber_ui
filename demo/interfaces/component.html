The Task interface defines the properties (and types) that Cyber UI cares about with regard regard to any given task. Most of the fields are optional. Consumers are encouraged to provide their own task interface which extends Task.

The CyberUiActionContext interface defines properties available for use by task action handlers. The initial use case is to expose a view container ref to action handlers, to allow, for example, attaching a menu next to the action button when the action button is clicked.

See the interface definition on GitHub.
